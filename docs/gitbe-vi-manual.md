## 三、标志组合规范

### 3.1 横向组合（主标志）

```
┌──────────────────────────────────────┐
│                                      │
│    [图标]  Gitbe                      │
│                                      │
└──────────────────────────────────────┘
```

**适用场景：**
- 网站头部导航
- 印刷品标题
- 社交媒体封面

### 3.2 纵向组合

```
┌──────────┐
│          │
│  [图标]  │
│          │
│  Gitbe   │
│          │
└──────────┘
```

**适用场景：**
- App 图标
- 名片
- 小尺寸展示

### 3.3 最小组合（图标版）

```
┌──────┐
│      │
│ [图标]│
│      │
└──────┘
```

**适用场景：**
- Favicon
- App 图标
- 社交媒体头像

### 3.4 组合间距规范

**图标与文字间距：**
- 横向组合：图标右侧留 1 个字母 'i' 的宽度
- 纵向组合：图标下方留 1 个字母 'G' 的高度

**示例：**
```
[图标]→(1×i)←Gitbe
```

## 四、标志禁用规范

### 4.1 禁止的修改

❌ **禁止拉伸变形**
- ❌ 拉宽或压扁标志
- ❌ 非等比缩放

❌ **禁止改变方向**
- ❌ 旋转标志
- ❌ 镜像翻转

❌ **禁止改变颜色**
- ❌ 使用非标准色彩
- ❌ 修改深色/浅色模式配色

❌ **禁止添加元素**
- ❌ 添加装饰性元素
- ❌ 添加阴影、发光等特效
- ❌ 添加文字说明

❌ **禁止拆分组合**
- ❌ 单独使用文字
- ❌ 单独使用图标（除非是最小组合场景）

### 4.2 禁用的使用方式

❌ **禁止在以下场景使用：**
- ❌ 低分辨率模糊显示
- ❌ 背景色冲突（色差小于 30%）
- ❌ 复杂图案背景
- ❌ 文字背景

## 六、标志安全空间

### 6.1 安全空间定义

安全空间是指标志周围必须保留的最小空白区域，确保标志清晰可读。

**安全空间大小：**
- 左右：标志宽度的 20%
- 上下：标志高度的 20%

**示意图：**
```
┌────────────────────────────────┐
│                                │
│    ┌────────────────────┐     │
│    │                    │     │
│    │  [图标]  Gitbe     │     │
│    │                    │     │
│    └────────────────────┘     │
│                                │
└────────────────────────────────┘
   ↑    ↑              ↑    ↑
20%   20%            20%   20%
```

### 6.2 安全空间应用

**安全空间内禁止：**
- ❌ 放置其他文字
- ❌ 放置图形元素
- ❌ 放置图片
- ❌ 放置其他标志

**例外情况：**
- ✅ 纯色背景可以延伸到安全空间
- ✅ 网格线可以经过安全空间（设计稿）
